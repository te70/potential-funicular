Timestamp Microservise

  1. The API endpoint is GET [url]/api/timestamp/:date_string?
  2. A date string is valid if can be successfully parsed by new Date(date_string).
  3. API accepts unix time
  4. If the date string is empty the services uses the current timestamp.
  5. If the date string is valid the api returns a JSON having the unix structure
  6. If the date string is Invalidthe api returns a JSON having the structure
    7. {"error" : "INvalid Date"}

Example Usage:

  • [url]/api/timestamp/2015-12-25
  • [url]/api/timestamp/1451001600000

Example Output:

{"unix":1451001600000, "utc":"Fri, 25 Dec 2015 00:00:00 GMT"}
